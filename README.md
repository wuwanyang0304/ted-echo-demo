# TED Echo

> 将内容观看转化为观点形成、表达、回应与成长记录的 AI 思辨表达原型。

[在线体验](https://wuwanyang0304.github.io/ted-echo-demo/) | [GitHub 仓库](https://github.com/wuwanyang0304/ted-echo-demo)

## 产品简介

TED Echo 面向“看完有想法，却不知道如何说清楚”的青年用户，以 TED 等全球议题内容为思想入口，通过多视角提示、短时表达、观点结构化、不同立场回应和思辨画像，帮助用户把碎片化感受逐步转化为立场、理由、例子与可继续讨论的问题。

产品遵循“AI 不替用户决定立场”的原则：用户先表达，AI 再进行结构化辅助；用户可以确认、修改或忽略 AI 结果，并自主决定对话、总结和画像的可见范围。

当前项目是用于赛事第一阶段展示产品流程与交互逻辑的概念原型，不是 TED 官方产品，也未使用或冒充真实 TED 演讲内容。

## 完整用户流程

### 1. 进入议题

用户查看原型议题、议题坐标，并记录观看前的初始判断，作为后续观点变化的参照。

### 2. 选择思考入口

用户可从支持、质疑、因果、经历、角色、未来六类视角中选择一个入口，也可以随机选择或跳过提示，直接自由表达。

### 3. 完成 60 秒 My Talk

用户通过文字或语音表达自己的观点。文字模式会展示 AI 观点预测卡；语音模式将发音反馈与观点质量分开，避免把语言表现等同于思辨能力。

### 4. 确认观点结构

系统将表达整理为“立场—理由—例子—问题”四部分。结果必须由用户确认或修改，AI 不替用户补写立场。

### 5. 选择真实观点回应

用户查看支持、质疑和经历向观点的分布，从种子观点中选择一条进行回应。正式版本将逐步替换为经用户授权的真实观点。

### 6. 生成讨论总结

系统整理双方表达中的共识、分歧、证据差异和新问题，不判断谁对谁错，也不生成虚假的“标准答案”。

### 7. 沉淀思辨成长记录

用户可以保存对话，并查看判断方式、观点依据、讨论姿态和表达主动性四个维度。画像结论附带表达证据，支持用户校准、关闭或删除；分享范围分为仅自己可见、平台匿名可见和身份公开可见。

## 当前哪些内容是模拟

本项目目前为纯前端交互原型，以下内容均用于演示产品逻辑，不代表真实业务结果：

| 模块 | 当前实现 | 正式版本规划 |
| --- | --- | --- |
| TED 议题 | 使用自拟原型议题和模拟时长 | 入围后使用企业确认的授权内容 |
| 参与人数 | “128 人参与”等为界面模拟数据 | 接入真实活动与课程参与数据 |
| AI 观点预测 | 前端规则和预设文本模拟 | 接入飞书 AI，根据用户原文生成并由用户确认 |
| 语音输入与发音评分 | 模拟录音状态和评分结果 | 接入语音转写及发音评测能力 |
| 观点分布 | 基于 24 条种子观点的模拟比例 | 基于经授权的真实用户观点统计 |
| 同伴观点 | 使用编号种子观点演示冷启动 | 逐步替换为匿名、授权的真实观点 |
| 讨论总结 | 使用预设共识、分歧和新问题 | 使用 AI 整理双方真实表达，允许用户修改 |
| 思辨画像 | 根据示例文本关键词计算演示刻度 | 基于多次表达证据更新，支持解释与校准 |
| 成长数据与徽章 | 次数、百分比和徽章均为模拟 | 根据真实完成记录计算 |
| 数据存储 | 仅使用浏览器本地存储 | 入围后设计账号、权限、删除和授权机制 |

当前 Demo 未连接大模型 API、飞书账号、真实社区、支付系统或生产数据库。

## 入围后如何接入飞书 AI

报名阶段优先验证交互闭环；获得赛事提供的飞书 AI 权限后，计划按以下顺序接入：

| 阶段 | 飞书能力规划 | 作用 | 产品边界 |
| --- | --- | --- | --- |
| 观点结构化 | 飞书多维表格 AI / AI 字段 | 从用户原文提取立场、理由、例子和问题 | 只依据原文，结果必须由用户确认 |
| 讨论总结 | 飞书 AI 文本生成能力 | 总结共识、分歧、证据差异和新问题 | 不判断对错，不生成虚假同伴观点 |
| 语音表达 | 飞书妙记等语音转写能力（以赛事开放权限为准） | 将 My Talk 转为可编辑逐字稿 | 无权限时保留文字输入，不阻塞核心闭环 |
| 观点与授权管理 | 飞书多维表格 | 保存观点类别、匿名编号、授权状态和回应关系 | 默认私密，分项授权，可撤回或删除 |
| 运营触达 | 多维表格自动化 / 消息能力 | 提醒用户查看回应、总结和新议题 | 用户可关闭，避免过度打扰 |
| 教学协作 | 飞书文档与群组协作 | 支持教师组织课前讨论和复盘 | 教师只能访问获得授权的内容 |

首个可落地版本将优先跑通“文字表达 → 观点结构化 → 真实观点回应 → 讨论总结”闭环，再扩展语音、课程活动和长期画像。

## 本地运行方法

本项目为单文件静态网页，无需安装前端依赖或执行构建命令。

### 方法一：直接打开

下载仓库后，双击 `index.html` 即可查看。

### 方法二：启动本地静态服务器

```bash
git clone https://github.com/wuwanyang0304/ted-echo-demo.git
cd ted-echo-demo
python -m http.server 8080
```

浏览器访问：

```text
http://127.0.0.1:8080/
```

也可以使用 VS Code 的 Live Server 打开 `index.html`。
